# Contributing to CeCell  
### Guidelines • Standards • Workflow • Mission Alignment

Thank you for your interest in contributing to **CeCell**, the accessibility‑first, voice‑driven, autonomous developer assistant built as part of the Ascendia ecosystem.  
CeCell is more than a VS Code extension — it is a mission to empower developers with clarity, speed, and accessibility.

This document outlines how to contribute effectively and responsibly.

---

# ❤️ Our Mission

CeCell is built on three pillars:

### **Accessibility**
Every feature must support low‑strain, voice‑driven, inclusive workflows.

### **Autonomy**
CeCell should help developers analyze, repair, and evolve software with minimal friction.

### **Clarity**
All code, documentation, and UX must be clean, structured, and easy to understand.

Contributions should reflect these values.

---

# 🧱 Code of Conduct

By contributing, you agree to:

- Be respectful and collaborative  
- Write clear, maintainable code  
- Prioritize accessibility in UI and workflows  
- Document your changes  
- Avoid breaking existing functionality  
- Uphold the mission of CeCell and Ascendia  

---

# 🛠️ How to Contribute

There are several ways to contribute:

### ✔ Add new commands  
### ✔ Improve existing commands  
### ✔ Add new panels or enhance UI  
### ✔ Improve the patch, merge, or analysis engines  
### ✔ Expand documentation  
### ✔ Fix bugs  
### ✔ Improve accessibility  
### ✔ Integrate new Ascendia Cloud features  

All contributions are welcome as long as they align with CeCell’s mission.
