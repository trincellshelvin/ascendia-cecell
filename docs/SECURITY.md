# CeCell Security Policy  
### Reporting • Handling • Expectations • Responsibilities

Security is a core pillar of CeCell and the Ascendia ecosystem.  
We take vulnerabilities seriously and appreciate responsible disclosure.

---

# 🔐 Reporting a Vulnerability

If you discover a security issue, please report it privately:

**Email:** security@ascendia.dev  
**Subject:** “CeCell Security Report”

Include:
- Description of the issue  
- Steps to reproduce  
- Potential impact  
- Suggested fix (optional)  

We will acknowledge your report within **48 hours**.

---

# 🛡️ Supported Versions

| Version | Supported |
|--------|-----------|
| 1.x.x  | ✔ Active support |
| 0.x.x  | ✖ No longer supported |

---

# 🧪 Vulnerability Handling Process

1. **Report received**  
2. **Issue triaged** (severity + scope)  
3. **Fix developed**  
4. **Fix tested**  
5. **Patch released**  
6. **Changelog updated**  
7. **Reporter notified**  

---

# 🔒 Security Principles

- No sensitive data stored locally  
- No cloud sync without explicit user consent  
- No telemetry without transparency  
- Panels sandboxed via VS Code Webview isolation  
- Commands validated before execution  
- Patch and merge engines run in controlled environments
